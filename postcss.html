A tool for transforming CSS with JavaScript

PostCSS was launched in 2013 by Andrey Sitnik, who also created autofixer. PostCSS was developed as a method to use JavaScript for CSS processing. PostCSS on it’s own is just an API, but when it’s used with one or more of it’s vast number of plugins, it has powerful abilities. It uses the Node.js framework, which makes it’s abilities easily modified and customized. Because PostCSS is an API, it allows users to create their own custom plugins and tools for features they might need. PostCSS isn’t a preprocessor per se, some people regard it as one, but other regard it as a postprocessor, or something else entirely. PostCSS will read your CSS with special additional syntax and will process it and return normal CSS code.

Features

Increase code readability

Automatically add vendor prefixes using the autoprefixer plugin.

// CSS input
:fullscreen {
}

// PostCSS output
:-webkit-full-screen {
}
:-ms-fullscreen {
}
:fullscreen {
}

Use tomorrow’s CSS today!

Use modern CSS features that may not be supported yet, the PostCSS Preset Env plugin can automatically polyfill these features.

// CSS input
body {
 color: lch(53 105 40);
}

// PostCSS
body {
 color: rgb(250, 0, 4);
}

The end of global CSS

Don't worry about naming your CSS classes, the CSS Modules plugin allows you to just use whatever name makes the most sense.

// CSS input
.name {
 color: gray;
}

// PostCSS output
.Logo__name__SVK0g {
 color: gray;
}

“I’m not compelled by PostCSS because I don’t love having to cobble together the processing features that I want. That also has the bad side effect of making the process of writing CSS different across projects. Plus, I don’t like the idea of preprocessing away modern features, some of which can’t really be preprocessed (e.g. custom properties cannot be preprocessed). But I did love Autoprefixer when we really needed that, which is built on PostCSS.”

Chris Coyier

Try this preprocessor

To start using PostCSS, choose a build tool that has an extension for PostCSS. Possible build tools include (but aren’t limited to) Gulp, Webpack, Parcel, CSS-in-JS and npm Scripts. On the PostCSS github page you can find a list of build tools, and which extension you need for them.

  • Choose a build tool
  • Find and add a PostCSS extension for it
  • Choose which plug-ins you want to use
  • Add them to your PostCSS process.
  • Start!
